# Протокол JSON-RPC 2.0

Описание протокола для внешних систем, интегрирующихся с API событий.

## Endpoint

```http
POST /api/rpc
Content-Type: application/json
```

## Формат запроса

```json
{
  "jsonrpc": "2.0",
  "method": "<имя метода>",
  "params": { ... },
  "id": <идентификатор запроса>
}
```

| Поле    | Тип                      | Обязательно | Описание                              |
|---------|--------------------------|-------------|---------------------------------------|
| jsonrpc | string                   | Да          | Версия протокола, должно быть `"2.0"` |
| method  | string                   | Да          | Имя вызываемого метода                |
| params  | object                   | Да          | Параметры метода                      |
| id      | number \| string \| null | Нет         | Идентификатор запроса                 |

## Формат ответа

### Успех

```json
{
  "jsonrpc": "2.0",
  "result": { ... },
  "id": <идентификатор из запроса>
}
```

### Ошибка

```json
{
  "jsonrpc": "2.0",
  "error": {
    "code": <код ошибки>,
    "message": "<сообщение>"
  },
  "id": <идентификатор из запроса>
}
```

## Коды ошибок

| Код    | Константа              | Причина                                   |
|--------|------------------------|-------------------------------------------|
| -32700 | Ошибка парсинга        | Невалидный JSON                           |
| -32600 | Некорректный запрос    | Неверная структура запроса или не `"2.0"` |
| -32601 | Метод не найден        | Метод не существует                       |
| -32602 | Некорректные параметры | Невалидные параметры метода               |
| -32603 | Внутренняя ошибка      | Внутренняя ошибка сервера                 |

## Методы

### event.create

Создаёт новое событие. ID события генерируется атомарно на стороне сервера.

**Параметры:**

| Параметр | Тип    | Обязательно | Описание                    | Диапазон         |
|----------|--------|-------------|-----------------------------|------------------|
| lat      | number | Да          | Широта местоположения       | от -90 до 90     |
| lng      | number | Да          | Долгота местоположения      | от -180 до 180   |
| oid      | number | Да          | Идентификатор объекта       | целое >= 1       |
| sp       | number | Да          | Скорость в км/ч             | от 0 до 300      |

**Пример запроса:**

```json
{
  "jsonrpc": "2.0",
  "method": "event.create",
  "params": {
    "lat": 55.751244,
    "lng": 37.618423,
    "oid": 42,
    "sp": 60
  },
  "id": 1
}
```

**Пример успешного ответа:**

```json
{
  "jsonrpc": "2.0",
  "result": {
    "id": 123
  },
  "id": 1
}
```

**Пример ответа с ошибкой:**

```json
{
  "jsonrpc": "2.0",
  "error": {
    "code": -32602,
    "message": "Некорректные параметры"
  },
  "id": 1
}
```

## HTTP статус

Endpoint всегда возвращает HTTP **200 OK**, даже при ошибках JSON-RPC.
Для определения успеха/ошибки необходимо анализировать поля `result` и `error` в теле ответа.
